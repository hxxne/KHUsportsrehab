# Antigravity Agent 작동 규칙 (Rules)

이 프로젝트에서 작업할 때 에이전트는 반드시 다음 규칙을 엄격히 준수해야 합니다.

1. 무조건 한국어로 답변할 것: 
   - 모든 생각(Thought)과 사용자 응답은 반드시 자연스러운 한국어로 작성할 것.
   - 변수명이나 코드 내의 영문은 그대로 두되, 주석이나 설명은 한국어를 사용할 것.

2. 안전성 우선 (사전 동의 필수):
   - 기존의 잘 돌아가는 코드를 대폭 수정하거나, 파일을 삭제하기 전에는 반드시 사용자에게 "이렇게 변경해도 될까요?"라고 물어보고 승인을 얻은 후 진행할 것.

3. 단계별 계획(Planning) 수립:
   - 복잡한 기능을 구현할 때는, 코드를 짜기 전에 먼저 Implementation Plan(구현 계획)을 마크다운 아티팩트로 정리하여 사용자에게 보여줄 것.

4. 친절하고 충분한 코드 설명 (교육용):
   - 사용자는 전문 개발자가 아닌 코딩 초보자(교육자)입니다. 코드를 작성하거나 수정할 때마다 단순히 코드만 던져주지 말고, 각 로직이 어떤 역할을 하는지 한글 주석과 함께 아주 상세히 설명해 줄 것.

5. 코드 생략 절대 금지 (Copy & Paste 보장):
   - 초보자는 코드가 생략되면 어디에 붙여넣어야 할지 모릅니다. 코드를 출력할 때 `// ... 기존 코드 ...` 형태로 생략하지 말고, 바로 복사해서 덮어쓸 수 있도록 수정된 전체 코드를 제공할 것.

6. 단계별 진행 (Step-by-Step):
   - 한 번에 너무 많은 파일이나 복잡한 기능을 한꺼번에 수정하지 말 것. 하나의 기능이 완벽하게 작동하는지 확인받은 후 다음 단계로 넘어갈 것.

7. Supabase 쿼리 관리 규칙:
   - 모든 SQL 쿼리 파일은 생성 시 파일명 앞에 `000_` 형태의 순서 번호를 반드시 붙여서 생성할 것 (예: `001_create_users_table.sql`).
   - 생성된 모든 쿼리 파일은 프로젝트 내에 흩어지지 않게 반드시 쿼리 전용 폴더(예: `supabase/migrations` 등) 안에 모아서 정리할 것.

8. 강력한 보안 및 암호화 (최우선 과제):
   - 개발 시 보안을 최우선으로 고려할 것. 
   - 사용자의 개인정보나 민감한 데이터(학생 정보, 비밀번호 등)를 다룰 때는 데이터베이스 전달 및 저장 전 반드시 암호화 로직을 적용할 것.
